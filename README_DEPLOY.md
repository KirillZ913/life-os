# Life OS — установка и синхронизация
*Три шага: опубликовать → установить → связать устройства. Суммарно ~30 минут, всё бесплатно.*

---

## Шаг A — Публикация на GitHub Pages (~10 мин)

1. Зарегистрируйся/войди на **github.com**.
2. Справа вверху **+** → **New repository**. Имя: `life-os`. Видимость: **Public** (обязательно для бесплатного Pages). → **Create repository**.
3. На странице пустого репозитория: **uploading an existing file** → перетащи ВСЕ пять файлов из архива (`index.html`, `manifest.webmanifest`, `sw.js`, `icon-192.png`, `icon-512.png`) → **Commit changes**.
4. **Settings** (вкладка репозитория) → слева **Pages** → Source: **Deploy from a branch** → Branch: **main**, папка **/ (root)** → **Save**.
5. Через 1–2 минуты вверху появится ссылка вида `https://ТВОЙ_НИК.github.io/life-os/` — это постоянный адрес Life OS. Открой и проверь.

**Обновления в будущем:** заменяешь `index.html` в репозитории (Add file → Upload, перезапись). Чтобы офлайн-кэш подтянул новую версию — в `sw.js` меняй строку `CACHE = "life-os-v2-6"` на новый номер.

## Шаг B — Установка как приложения (~5 мин)

**MacBook (Chrome):** открой адрес → в адресной строке справа значок установки (монитор со стрелкой) → **Установить**. Life OS появится в Dock/Launchpad отдельным окном без браузерной обвязки.
**MacBook (Safari):** адрес → меню **Файл → Добавить в Dock**.

**iPhone (Safari):** открой адрес → кнопка **Поделиться** (квадрат со стрелкой) → **На экран «Домой»** → Добавить. Иконка-печать встанет среди приложений, запуск — полноэкранный, работает офлайн.

**Важно про данные:** localStorage привязан к адресу. Твои данные из локального файла сюда не переедут сами — в старом файле сделай **Основа → Данные → Экспорт JSON**, а в установленном приложении **Импорт**. Один раз.

## Шаг C — Синхронизация MacBook ↔ iPhone (~15 мин)

Работает через бесплатный Supabase (твоя личная база, никто кроме тебя доступа не имеет).

1. **supabase.com** → Sign up (можно через GitHub) → **New project**: имя `life-os`, пароль базы — любой (сохрани), регион — Frankfurt/ближайший → Create. Подожди ~2 мин.
2. Слева **SQL Editor** → New query → вставь целиком и нажми **Run**:

```sql
create table if not exists life_os (
  id text primary key,
  data jsonb not null,
  updated_at timestamptz not null default now()
);

alter table life_os enable row level security;

create policy "token-access" on life_os
  for all
  using  (id = (current_setting('request.headers', true)::json->>'x-sync-id'))
  with check (id = (current_setting('request.headers', true)::json->>'x-sync-id'));
```

Это создаёт хранилище и правило: строку видит только тот, кто знает её код связки. Даже с публичным ключом чужие данные не читаются.

3. Слева **Project Settings → API**. Скопируй два значения:
   - **Project URL** (вида `https://xxxx.supabase.co`)
   - **anon public** ключ (длинный, начинается с `eyJ`)
4. **В Life OS на MacBook:** Основа → **Синхронизация · облако** → вставь URL и ключ → **Сгенерировать** код связки → **Сохранить** → **Отправить ↑**. Появится «Отправлено в облако ✓».
5. **Скопируй код связки** (24 символа) — перешли себе любым способом.
6. **На iPhone** в установленном Life OS: та же секция → тот же URL, тот же ключ, **вставь код связки** → Сохранить → **Забрать ↓**. Всё: данные на телефоне.
7. Включи режим **Авто** на обоих устройствах — изменения будут уезжать в облако сами через несколько секунд после правок, а при запуске приложение будет подтягивать свежее.

**Логика конфликтов:** последняя запись побеждает. При ручном «Забрать» приложение показывает дату облачной копии и спрашивает подтверждение.

## Если что-то не так

- Пустая страница после установки → провентилируй кэш: удали приложение с экрана, открой адрес в браузере, поставь заново.
- «Не отправилось» → проверь, что URL без хвостового `/`, ключ — именно **anon public**, SQL из шага C.2 выполнен.
- Данные разъехались → на «правильном» устройстве Отправить ↑, на втором Забрать ↓.
- Бэкап никто не отменял: **Экспорт JSON раз в неделю** — привычка, даже с облаком.
